# EasyX Flight Chess / 飞行棋游戏

一个基于 **C++、Visual Studio 2022 和 EasyX 图形库** 编写的图形化飞行棋小游戏。项目以传统飞行棋规则为基础，加入了正六边形棋盘、多人配置、鼠标交互、特殊格、返回上一步和 GBK 日志记录等功能，适合作为 C++ 课程实践、图形界面练习和游戏逻辑设计示例。

## 2. 项目简介

本项目实现了一个可鼠标操作的图形化飞行棋游戏。游戏左侧为正六边形棋盘，右侧为信息面板和操作区域。玩家可以在开始界面选择 **2-6 名玩家**，并为每名玩家设置 **2-6 个棋子**。进入游戏后，系统先通过抽签决定行动顺序，再按照掷骰、选择棋子、移动、触发特殊格、判断胜利的流程循环运行。

项目整体采用面向对象方式组织代码：`game` 类负责游戏主流程，`board` 类负责棋盘生成、绘制和移动计算，`player` 与 `piece` 负责玩家和棋子状态管理，`ui` 文件负责按钮、文字、骰子、棋子等界面绘制工具。

## 3. 已实现功能

### 3.1 基础游戏功能

- 支持 2-6 名玩家参与游戏。
- 每名玩家支持 2-6 个棋子。
- 支持红、蓝、绿、黄、紫、橙六种玩家颜色。
- 开局抽签决定行动顺序，点数高者优先，同点数按玩家编号靠前者优先。
- 掷出 6 才能让基地内棋子起飞。
- 掷出 6 后，本次行动结束后可额外再投一次。
- 棋子可以从基地、公共跑道、终点列移动到停车区。
- 率先让所有棋子进入终点列或停车区的玩家获胜。

### 3.2 图形界面功能

- 使用 EasyX 绘制图形化窗口。
- 左侧显示正六边形公共跑道、玩家基地、终点列和中心停车区。
- 右侧显示当前操作说明、骰子点数、特殊格提示、玩家进度和规则说明。
- 全程使用鼠标点击完成操作，无需键盘输入。
- 设置界面、抽签界面、游戏界面和结束界面分状态显示。
- 可移动棋子会在选择阶段进行标记提示。
- 多个棋子落在同一格时采用错位显示，避免完全重叠。

### 3.3 特殊规则功能

- 奖励格：棋子落到指定格子后前进若干格。
- 惩罚格：棋子落到指定格子后后退若干格。
- 黑色陷阱格：同一棋子本圈第二次踩中后需要额外多跑一圈。
- 对方起点格：棋子落到其他玩家起点时会暂停一回合。
- 同色普通格：棋子落到自己颜色普通格时，可以飞跃到下一个同色普通格。
- 终点列判断：到达终点入口后进入对应玩家的终点列，步数超出范围时判定为不可移动。

### 3.4 辅助功能

- 支持“返回上一步”，通过快照栈保存最近的游戏状态。
- 每局游戏自动生成日志文件。
- 日志保存路径为 `result/` 文件夹。
- 日志文件名格式为 `YYYYMMDD_HHMMSS.txt`。
- 日志使用 GBK 编码，记录抽签、投骰、移动、特殊格、返回上一步和胜利结果。

## 7. 类与对象关系

### 7.1 `main.cpp`

`main.cpp` 是程序入口，主要负责：

- 初始化 EasyX 图形窗口。
- 设置窗口标题。
- 创建 `game` 对象。
- 调用 `game::init()` 完成初始化。
- 调用 `game::run()` 进入游戏主循环。
- 游戏退出后关闭图形窗口。

### 7.2 `game` 类

`game` 类是整个项目的核心控制类，主要负责：

- 管理游戏状态：设置、抽签、投骰、选择棋子、游戏结束。
- 保存玩家列表、行动顺序、骰子点数和胜利者信息。
- 处理鼠标点击事件。
- 调用棋盘与 UI 绘制函数刷新界面。
- 判断是否有棋子可移动。
- 执行棋子移动、跳过、切换回合和胜负判断。
- 维护历史快照栈，实现“返回上一步”。
- 生成和写入 GBK 游戏日志。

### 7.3 `board` 类

`board` 类负责棋盘相关逻辑，主要包括：

- 构建 60 格公共跑道。
- 生成正六边形棋盘坐标。
- 生成每名玩家的起点、基地、终点列和停车区。
- 标记奖励格、惩罚格、黑色陷阱格和玩家起点格。
- 绘制棋盘、棋子、基地和终点列。
- 计算棋子移动后的公共跑道位置或终点列位置。
- 判断鼠标点击命中了哪个棋子。
- 查找下一个同色普通格。

### 7.4 `player` 与 `piece`

`player` 表示玩家，保存玩家颜色、名称、起点、终点入口和棋子列表。

`piece` 表示单个棋子，保存棋子编号、所属玩家、当前位置状态、公共跑道位置、终点列位置、暂停回合数、陷阱计数和额外圈数。

### 7.5 `ui` 工具函数

`ui.h` 和 `ui.cpp` 负责界面绘制工具，包括：

- 按钮类 `button`。
- 文字居中、左对齐和自动换行绘制。
- 六边形绘制。
- 棋子圆形绘制。
- 骰子绘制。
- 标记符号绘制。
- 玩家名称与颜色辅助函数。

## 8. 主要算法与实现思路

### 8.1 游戏状态机

游戏流程使用 `gamestate` 枚举进行管理：

```cpp
STATE_SETUP   // 设置人数和棋子数
STATE_LOT     // 抽签决定出发顺序
STATE_ROLL    // 等待当前玩家投骰子
STATE_SELECT  // 等待当前玩家选择棋子或不走
STATE_OVER    // 游戏结束
```

程序根据当前状态分发鼠标点击事件，并绘制对应界面。这样可以避免把所有操作逻辑写在同一个函数里，使流程更清晰，也方便后续扩展暂停、存档、重新开始等功能。

### 8.2 正六边形棋盘生成

公共跑道共有 60 格，被平均分布在正六边形的 6 条边上，每条边 10 格。程序先根据圆心、半径和角度计算六边形 6 个顶点，再通过线性插值生成每条边上的格子中心坐标。

这种做法的优点是棋盘不是手工写死坐标，而是由数学公式自动生成，后续如果要调整棋盘大小、位置或格子数量，会更容易维护。

### 8.3 起点与终点列计算

不同玩家人数下，每名玩家的起点按照以下方式等距分布：

```cpp
start = player_id * TRACK_COUNT / numplayers;
```

每名玩家的终点入口为起点前一格：

```cpp
endpos = (startpos + TRACK_COUNT - 1) % TRACK_COUNT;
```

当棋子移动跨过终点入口时，`board::move()` 会将结果转换为终点列位置；如果步数超过终点列长度，则判定本次移动无效。

### 8.4 掷骰与行动顺序

抽签阶段，每名玩家点击骰子得到一个 1-6 的随机数。抽签完成后使用 `std::sort` 对玩家编号进行排序：

- 点数大的玩家先行动。
- 点数相同的情况下，玩家编号小者先行动。

正式游戏阶段同样通过随机数模拟骰子点数，并根据是否掷出 6 判断能否起飞和是否获得额外回合。

### 8.5 特殊格处理

特殊格使用 `std::map<int, int>` 记录：

- `rewardmap`：奖励格位置和前进步数。
- `punishmap`：惩罚格位置和后退步数。
- `trapmap`：黑色陷阱格位置。

棋子完成普通移动后，程序根据当前位置依次判断是否触发奖励、惩罚、陷阱、对方起点或同色飞跃。特殊效果集中在 `game::apply_special()` 中处理，逻辑相对集中，方便修改规则。

### 8.6 返回上一步

项目通过 `snapshot` 结构体保存游戏完整状态，包括玩家、棋子、棋盘、骰子、当前状态、行动顺序和提示文字等。每次关键点击前保存一次快照，点击“返回上一步”时从历史栈中恢复。

为了避免内存无限增长，历史记录最多保留 80 步。

### 8.7 日志记录

每局游戏开始时，程序会在 `result/` 文件夹中创建日志文件。日志按当前时间命名，并通过 `WideCharToMultiByte(936, ...)` 将宽字符串转换为 GBK 编码写入文本文件。

日志记录便于复盘游戏过程，也方便课程作业或项目展示时证明程序运行过程。

## 9. 变量命名与代码风格

本项目变量命名以直观、短小为主，整体保持 C++ 课程项目常见风格。

- 类名：使用小写英文名，如 `game`、`board`、`button`。
- 结构体名：使用小写英文名，如 `piece`、`player`、`snapshot`。
- 成员变量：使用语义化英文，如 `numplayers`、`curplayer`、`dicevalue`、`turnindex`。
- 坐标变量：使用 `x`、`y`、`cx`、`cy` 表示左上角或中心点坐标。
- 状态变量：使用 `state`、`hasrolled`、`gotsix`、`winner` 等表示游戏状态。

代码中保留了较多中文注释，便于课程检查、答辩展示和后续维护。

## 11. 未完成内容与后续可完善方向

当前项目已经可以运行和演示，但仍存在一些可以继续完善的地方：

1. **击退规则接口尚未真正实现**
   代码中保留了 `game::check_hit()` 接口，但当前实现为空。也就是说，当前版本并没有真正完成“落到对方棋子所在格后把对方击回基地”的规则，而是主要通过同格错位显示避免视觉重叠。后续可以在该接口中补充击退逻辑。
2. **缺少存档与读档功能**
   当前只能通过日志记录过程，不能把一局游戏保存后下次继续。
3. **缺少人机玩家**
   当前所有玩家均由本地鼠标操作，暂未实现电脑自动决策。
4. **规则仍可进一步细化**
   不同版本飞行棋规则存在差异，例如是否必须刚好进入终点、撞子规则、叠子规则等，后续可以将规则设计成可配置选项。
5. **跨平台能力有限**
   项目依赖 EasyX 和 Windows API，暂不支持 Linux、macOS 或 Web 端运行。
6. **工程发布结构可继续清理**
   GitHub 仓库建议只保留源码和必要说明，不提交本地编译缓存、VS 临时文件和二进制调试文件。

## 12. 预留开发接口

项目中已经有一些适合后续扩展的接口或模块：

- `game::check_hit()`：可用于实现撞子、击退、叠子等规则。
- `game::save()` / `game::undo()`：已用于返回上一步，可扩展为正式存档系统。
- `game::log_event()`：可扩展为更详细的日志系统或回放系统。
- `board::build_special()`：可集中修改奖励格、惩罚格、陷阱格和起点规则。
- `board::build_track()`：可扩展为不同形状或不同格数的棋盘。
- `ui.cpp` 绘图函数：可继续优化按钮样式、动画、提示框和主题皮肤。
